Warm-started higher order randomized SVD (rHOSVD) on GPUs for streaming tensor data
🔄 In Progress
StatusOngoing research RolePrincipal investigator

Overview

This project develops warm-started randomized higher-order SVD algorithms optimized for GPUs to efficiently process streaming tensor data. The goal is to maintain low-rank Tucker decompositions of high-dimensional tensors with minimal latency and memory overhead, leveraging prior solutions to accelerate convergence on new data arrivals.

GitHub Presentation

Motivation

  • Streaming tensor data arises in time-evolving physical systems, multi-channel sensor arrays, and high-dimensional time-series applications.
  • Traditional batch SVD methods incur prohibitive costs when recomputing from scratch on each new data arrival.
  • Warm-started algorithms exploit prior solutions to dramatically reduce iterations and wall-clock time on GPU hardware.

Contributions (planned/in progress)

  • Algorithmic: warm-start strategy for randomized HOSVD exploiting prior Tucker factors.
  • Implementation: CUDA kernels for efficient tensor contractions and randomized orthogonalization on streaming buffers.
  • Evaluation: benchmarks on real and synthetic tensor streams; comparison with batch and incremental baselines.

Methods & technical approach

  • Tensor framework: Tucker decomposition; unfolding-based SVD on mode-$n$ matricizations.
  • Randomization: sketching with Gaussian or Rademacher test matrices; power iteration refinement.
  • Warm-start: reuse prior factors as initial guesses; adaptive rank updates as new data arrives.
  • GPU kernels: batched Gram matrix formation, QR factorization, and tensor reconstruction.

Current status

Research is actively underway. The repository is available on GitHub, and initial experimental results have been presented. More detailed write-ups and public sharing of code will follow as the project matures.
